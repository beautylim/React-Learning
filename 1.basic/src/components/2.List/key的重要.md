# Key的重要性

<mark>key</mark>是React用来识别列表中各个元素的身份证，当列表数据发生变化(增删改排序)时，React的协调算法（Reconciliation）会通过<kbd>Key</kbd>来高效地对比新旧两颗虚拟<u>DOM</u>树。它会根据Key来判断哪些元素是新创建的，哪些是被删除了，哪些只是移动了位置。
* **一个稳定且唯一** 的 <kbd>key</kbd> 能版主React最大限度地复用已有地<u>DOM</u>元素和组件实例，从而极大地提升性能
* 如果**不提供<kbd>key</kbd>**, React在控制台会给出警告
* **使用数组地索引作为<kbd>key</kbd>** 是一种常见地反模式, 应当激励避免。因为当列表项地顺序发送改变时（eg. 在数组开头插入一个新元素）, 所有后续元素的索引都会改变。这会让React误以为时元素自身内容发生了大规模变化，从而导致不必要的重新渲染，甚至丢失组件内部的状态
